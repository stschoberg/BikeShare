Los Angeles Bike Share Data Analysis

This site uses various Python libraries and Google APIs to analyze recent data from an LA bikeshare network. The data is availible here for further analysis. Click on the charts to learn more!

Data Visualization: Ride Duration

×

The dataset provides information about the duration of each ride in seconds. The code translates this data into minutes then plots the frequency of the rides lengths. While the mean-average ride lasts 25.92 minutes, a better indicator of the average, the median, is 10.0 minutes. Ride duration data is an important factor to consider when planning to upkeep the bike network. The company will want to ensure there are enough bikes to service all their customers in a given day. The longer someone is on a bike, the longer that bike is unavailible to others.

Average distance (km) traveled?

×

The dataset provides information about the length of each ride. his histogram shows the distribution of the durations of the rides in km. The mean-average ride lasts 1.53km where the median ride lasts 1.38km. Distance traveled per ride is an indicator of when the bikes will need to be serviced ( i.e air in tires, oil on gears). Having this data availible will aid the company in keeping the network functional.
